# Day 9 검증문제와 답변가이드

## 먼저 풀어볼 문제

1. config는 무엇인가?
2. 환경 변수는 왜 사용하는가?
3. DB 주소를 코드에 직접 쓰면 어떤 문제가 생기는가?
4. `main.go`는 어떤 역할을 하는가?
5. `PORT`는 기본값을 사용해도 될 수 있지만 `DATABASE_URL`은 보통 필수인 이유는 무엇인가?
6. Phase 2에서 `BLOCKCHAIN_RPC_URL`이 필요한 이유는 무엇인가?
7. 설정 누락은 요청 처리 중에 발견하는 것보다 시작 시점에 발견하는 것이 왜 좋은가?
8. config 패키지를 만들면 테스트에는 어떤 장점이 있는가?

## 답변가이드

### 1. config의 의미

config는 애플리케이션 실행에 필요한 설정값입니다.

### 2. 환경 변수를 사용하는 이유

환경마다 다른 값을 코드 수정 없이 주입하기 위해서입니다.

### 3. DB 주소를 코드에 직접 쓰는 문제

로컬, 테스트, 운영 환경을 바꿀 때마다 코드를 수정해야 합니다.

보안상 민감한 값이 코드에 남을 위험도 있습니다.

### 4. main.go의 역할

`main.go`는 설정, DB, repository, service, handler를 조립하고 서버를 시작하는 진입점입니다.

### 5. PORT와 DATABASE_URL 차이

`PORT`는 기본값으로 `8080`을 사용할 수 있습니다.

하지만 `DATABASE_URL`이 없으면 DB 연결 자체가 불가능하므로 서버 시작을 막는 편이 안전합니다.

### 6. BLOCKCHAIN_RPC_URL이 필요한 이유

Event Indexer나 Withdrawal broadcast가 블록체인 네트워크와 통신하려면 RPC 주소가 필요합니다.

### 7. 시작 시점에 설정 누락을 발견하는 이유

서버가 뜬 뒤 요청을 받을 때 실패하면 장애 원인을 늦게 알게 됩니다.

시작 시점에 실패하면 배포나 실행 단계에서 바로 문제를 발견할 수 있습니다.

### 8. 테스트 장점

config 구조체를 직접 만들어 테스트에 주입할 수 있으므로 환경 변수에 덜 의존하는 테스트를 작성할 수 있습니다.
