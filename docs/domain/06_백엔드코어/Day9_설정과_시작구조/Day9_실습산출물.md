# Day 9 실습산출물 - 설정 로딩과 애플리케이션 시작 구조

관련 Jira: [SPN-26](https://aslan0.atlassian.net/browse/SPN-26)

이 문서는 직접 채우는 실습산출물입니다.

정답 예시는 `Day9_실습가이드.md`와 `Day9_검증문제_답변가이드.md`에서 확인하고, 이 문서에는 먼저 본인이 이해한 내용을 적습니다.

작성 원칙:

```text
1. 먼저 실습가이드를 따라 코드 작업을 진행한다.
2. 코드에서 직접 확인한 내용을 이 문서에 적는다.
3. 정답처럼 완벽히 쓰려 하지 말고, 내가 이해한 말로 적는다.
4. 모르는 부분은 비워두지 말고 "무엇이 헷갈리는지"를 적는다.
```

## 1. main.go 실행 순서

아래 순서를 실제 `cmd/api/main.go`를 읽고 프로젝트 기준으로 채운다.

작성 방법:

```text
cmd/api/main.go를 위에서 아래로 읽으면서 "서버가 실행되기 전에 어떤 준비를 하는지"를 적는다.
코드에서 함수 호출이나 변수 생성이 보이면, 그 동작이 어떤 준비 단계인지 내 말로 바꿔 적는다.
```

| 순서 | 코드에서 확인한 동작 | 관련 파일/함수 | 내가 이해한 의미 |
| --- | --- | --- | --- |
| 1 | 어떤 설정값을 읽거나 만든다 | 코드에서 확인한 함수/변수 이름 | 이 단계가 왜 먼저 필요한지 적기 |
| 2 | 어떤 외부 자원을 연결한다 | 코드에서 확인한 함수/변수 이름 | 실패하면 서버가 어떻게 되는지 적기 |
| 3 | 어떤 HTTP 처리 객체를 만든다 | 코드에서 확인한 함수/변수 이름 | 요청을 나눠 보내기 위해 필요한지 적기 |
| 4 | 어떤 route를 등록한다 | 코드에서 확인한 함수/변수 이름 | 어떤 API가 연결되는지 적기 |
| 5 | 서버를 시작한다 | 코드에서 확인한 함수/변수 이름 | 어떤 주소/포트로 실행되는지 적기 |

관찰 메모:

```text
main.go에서 가장 먼저 눈에 들어온 코드:

아직 이해가 안 되는 코드:

Java/Spring Boot와 비교했을 때 비슷하다고 느낀 부분:
```

## 2. 현재 필요한 설정값

작성 방법:

```text
현재 main.go 또는 새로 만든 config.go에서 환경 변수로 읽는 값을 찾는다.
그 값이 없으면 서버가 실행될 수 있는지, 기본값으로 대체해도 되는지 판단한다.
```

| 설정 | 필수 여부 | 기본값 가능 여부 | 이유 |
| --- | --- | --- | --- |
| 코드에서 찾은 설정 이름 | 필수/선택/판단 어려움 | 가능/불가능/판단 어려움 | 그렇게 판단한 이유 |
| 코드에서 찾은 설정 이름 | 필수/선택/판단 어려움 | 가능/불가능/판단 어려움 | 그렇게 판단한 이유 |
| 추가로 필요해 보이는 설정 이름 | 필수/선택/판단 어려움 | 가능/불가능/판단 어려움 | 그렇게 판단한 이유 |

판단 기준 메모:

```text
필수 설정이라고 생각한 이유:

기본값을 둘 수 있다고 생각한 이유:

아직 판단이 어려운 설정:
```

## 4. config 구현 후보

작성 방법:

```text
Day9 실습가이드에서 만든 config 패키지를 기준으로 적는다.
실제로 만든 파일 경로, 구조체 이름, 함수 이름을 적고, 왜 그렇게 나눴는지 적는다.
```

```text
패키지 위치: 내가 만든 config 파일 경로
구조체 이름: 설정값을 담는 struct 이름
함수 후보: 설정값을 읽어오는 함수 이름
```

고민할 점:

```text
config가 어디까지 책임져야 한다고 생각하는가?

설정 누락 에러는 어떤 메시지로 보여주는 것이 좋다고 생각하는가?
```

## 5. 설정 누락 처리 기준

작성 방법:

```text
설정값이 없을 때 서버를 바로 종료해야 하는지, 기본값으로 실행해도 되는지, 특정 기능을 사용할 때만 막아도 되는지 나눠 적는다.
```

| 구분 | 설정 후보 | 이유 |
| --- | --- | --- |
| 서버 시작 시 반드시 실패해야 하는 설정 | 설정 이름 | 없으면 왜 위험한지 |
| 기본값을 사용해도 되는 설정 | 설정 이름 | 기본값으로 실행해도 되는 이유 |
| 특정 기능 실행 시점에 검증해도 되는 설정 | 설정 이름 | 전체 서버가 아니라 특정 기능에서만 필요한 이유 |
