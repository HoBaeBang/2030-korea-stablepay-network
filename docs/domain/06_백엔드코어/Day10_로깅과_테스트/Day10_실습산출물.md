# Day 10 실습산출물 - 로깅과 테스트 패턴

관련 Jira: [SPN-27](https://aslan0.atlassian.net/browse/SPN-27)

이 문서는 직접 채우는 실습산출물입니다.

로그 후보와 테스트 후보를 미리 정답처럼 채워두지 않습니다. 먼저 본인이 프로젝트 코드를 읽고 필요하다고 판단한 내용을 적은 뒤, `Day10_검증문제_답변가이드.md`와 비교합니다.

작성 원칙:

```text
1. 먼저 실습가이드를 따라 테스트 코드를 읽고 수정한다.
2. 테스트가 어떤 규칙을 막아주는지 내 말로 적는다.
3. 로그 후보는 "운영 중 문제가 생겼을 때 무엇을 알아야 하는가" 기준으로 적는다.
4. 모르는 부분은 비워두지 말고 "어떤 판단이 어려운지"를 적는다.
```

## 5. given / when / then 테스트 패턴

작성 방법:

```text
given에는 테스트 시작 상태를 쓴다.
when에는 실행할 함수나 요청을 쓴다.
then에는 기대하는 결과와 바뀌면 안 되는 값을 쓴다.
```

```text
테스트 이름: 어떤 조건이면 어떤 결과가 나와야 하는가
given: 테스트 시작 전에 어떤 데이터/상태가 준비되어 있는가
when: 어떤 함수를 어떤 요청값으로 실행하는가
then: 어떤 결과가 나와야 하고, 어떤 값은 바뀌면 안 되는가
```
