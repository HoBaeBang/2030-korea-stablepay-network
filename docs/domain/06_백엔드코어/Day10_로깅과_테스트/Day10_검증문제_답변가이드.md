# Day 10 검증문제와 답변가이드

관련 Jira: [SPN-27](https://aslan0.atlassian.net/browse/SPN-27)

## 먼저 풀어볼 문제

1. 로그는 왜 필요한가?
2. 로그와 API 에러 응답은 무엇이 다른가?
3. 돈과 상태가 바뀌는 순간에 로그가 중요한 이유는 무엇인가?
4. 로그에 private key를 남기면 안 되는 이유는 무엇인가?
5. Go 테스트 파일 이름은 보통 어떤 규칙을 따르는가?
6. 한글 subtest를 쓰면 어떤 장점이 있는가?
7. given/when/then은 각각 무엇을 의미하는가?
8. Ledger 구현 전에 테스트 패턴을 먼저 정리하면 어떤 장점이 있는가?

## 답변가이드

### 1. 로그가 필요한 이유

운영 중인 시스템에서 어떤 일이 발생했는지 추적하기 위해 필요합니다.

### 2. 로그와 API 에러 응답의 차이

API 에러 응답은 사용자에게 실패 이유를 알려주는 것입니다.

로그는 개발자와 운영자가 내부 상태를 추적하기 위한 기록입니다.

### 3. 돈과 상태 변경 로그가 중요한 이유

결제, 입금, 출금, 정산은 장애가 발생했을 때 정확한 추적이 필요합니다.

상태 변경 로그가 없으면 어디서 잘못되었는지 확인하기 어렵습니다.

### 4. private key를 로그에 남기면 안 되는 이유

private key가 노출되면 자산을 직접 이동시킬 수 있는 권한이 노출되는 것과 같습니다.

### 5. Go 테스트 파일 이름

보통 `_test.go`로 끝납니다.

### 6. 한글 subtest 장점

테스트가 도메인 문서처럼 읽히고, 어떤 케이스를 검증하는지 빠르게 이해할 수 있습니다.

### 7. given/when/then

`given`은 주어진 상황, `when`은 실행하는 행동, `then`은 기대 결과입니다.

### 8. Ledger 전 테스트 패턴 정리의 장점

Ledger는 돈의 이동 기록을 다루므로 실수가 치명적입니다.

테스트 패턴을 먼저 정리하면 이후 구현을 더 안전하게 진행할 수 있습니다.
