# Phase 2 첫 구현 범위 결정과 Sprint 2 백로그 기초 학습

관련 Jira: [SPN-22](https://aslan0.atlassian.net/browse/SPN-22)

이 문서는 Phase 2 Day 5 학습 허브입니다.

Day 5의 목표는 Day 1~4에서 학습한 도메인을 바탕으로 Phase 2의 첫 구현 범위를 결정하고, Sprint 2 백로그 후보를 정리하는 것입니다.

## 오늘의 큰 그림

![SPN-22 Day 5 학습 흐름](../confluence/diagrams/spn22-day5-learning-flow.png)

Day 5는 새로운 도메인을 많이 추가로 배우는 날이 아닙니다. 지금까지 배운 내용을 바탕으로 "무엇을 먼저 만들 것인가"를 결정하는 날입니다.

## 오늘의 목표

1. Day 1~4의 핵심 도메인을 한 번에 다시 설명할 수 있다.
2. Phase 2 첫 구현 후보를 비교할 수 있다.
3. Backend Core를 먼저 정리해야 하는 이유를 설명할 수 있다.
4. Sprint 2 백로그 후보를 문서로 정리할 수 있다.
5. Day 6 통합 점검으로 넘어가기 위한 질문 목록을 만든다.

## 퇴근 후 작업의 원칙

퇴근 후 작업은 사용자가 직접 진행합니다.

1. `docs/domain/phase-2-study-summary.md` 파일을 채운다.
2. Day 1~4에서 배운 핵심 도메인을 요약한다.
3. Phase 2 첫 구현 후보 3개를 비교한다.
4. 추천 첫 구현 작업 1개를 선택한다.
5. Sprint 2 백로그 후보를 작성한다.
