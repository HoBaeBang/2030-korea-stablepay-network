# 문서 인덱스

이 문서는 GitHub Markdown 문서와 Confluence 문서가 서로 어긋나지 않도록 관리하기 위한 인덱스입니다.

## 문서 관리 원칙

| 위치 | 역할 |
| --- | --- |
| GitHub Markdown | 원본 문서, 변경 이력, 코드와 함께 관리되는 기준 문서 |
| Confluence | 태블릿/모바일에서 읽기 좋은 학습 문서와 Jira 연결 문서 |
| Jira | 작업 단위, 상태, 완료 기준, 관련 문서 링크 관리 |

앞으로 새 문서를 만들 때는 가능하면 다음 순서를 따릅니다.

1. GitHub Markdown 원본을 먼저 만든다.
2. 필요한 경우 Confluence에 같은 내용을 읽기 좋게 옮긴다.
3. Jira 티켓에는 Confluence 링크와 GitHub 원본 링크를 함께 남긴다.
4. 다이어그램은 SVG를 원본으로 보관하고, Confluence 표시용은 PNG를 사용한다.
